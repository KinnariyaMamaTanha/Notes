如[[Insertion sort]]中的方法，每次都会增加一个被排序的数据

# **分治**法
当问题是递归的时候，可以将该问题拆分为更小的某几个子问题，**分而治之**，直到遇到基础情况（可以使用简单步骤解决的情况），再把子问题的答案合并为原问题的答案，如[[Merge sort]]

一般来说，分治法的时间消耗可以表示为：
$$
T(n) = 
\begin{cases}
\Theta(1) \quad n \le c\\
\sum_{i=1}^mT(n_i) + T(others) \quad n > c
\end{cases}
$$
其中 c 是一个固定的常数，$T(n_i)$表示第 i 种子情况所需时间，$T(others)$表示其他步骤所需时间。也可以使用递归树来计算
